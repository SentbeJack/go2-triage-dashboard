# GO2 Triage 대시보드 배포 가이드 이 저장소의 `index.html`은 Slack 자동 수신을 지원하는 GO2 Triage 대시보드예요. GitHub Pages에 배포하면 Google Sheets 백엔드와 실시간으로 통신할 수 있어요. ## 배포 방법 1. 이 저장소를 Private으로 본인 계정에 생성해요. 2. 저장소의 **Settings → Pages** 메뉴로 가요. 3. **Source** 드롭다운에서 `main` 브랜치를 선택하고 **Save**를 눌러요. 4. 배포가 완료되면 `https://본인계정.github.io/저장소명` URL이 생기지만, Private 저장소라 접근할 수 없어요. 5. 대신 **Settings → Pages** 하단의 "GitHub Pages visibility" 옵션을 "Private"으로 전환하고 Save하면 인증된 사용자만 접근 가능한 GitHub Pages URL이 생겨요. ## 접근 권한 설정 대시보드 URL은 인증된 GitHub 사용자만 접근할 수 있어요. 동료와 공유하려면: 1. 저장소의 **Settings → Collaborators** 메뉴에서 **Add people** 버튼을 눌러요. 2. 함께 쓸 동료의 GitHub 계정명을 입력하고 **Add 계정명 to 저장소명**을 눌러요. 3. 초대 메일이 전송되고, 상대방이 수락하면 접근 권한이 생겨요. URL 직접 공유보다는 저장소 접근 권한을 주는 게 관리상 편해요. 퇴사 등으로 권한 회수가 필요하면 **Settings → Manage access**에서 제거하면 되거든요. ## 백엔드 설정 대시보드는 Google Apps Script로 만든 백엔드와 통신해요. 스크립트 프로젝트는 별도의 시트에 만드는데, 가이드에 따라 스크립트를 배포하고 발급된 URL을 대시보드 설정에 입력하면 연결이 완료돼요. Slack 자동 수신도 동일한 백엔드로 이뤄지므로, Slack 앱 설정의 Request URL 항목에도 스크립트 URL을 넣어야 해요. ## 업데이트 대시보드 코드를 고칠 때마다 GitHub에 Push하면 자동으로 GitHub Pages에 배포돼요. Push 직후 잠시 캐시로 인해 변경이 반영되지 않을 수 있는데, 브라우저 강력 새로고침(Windows: Ctrl+F5, Mac: Cmd+Shift+R)으로 최신 버전을 불러올 수 있어요. ## 비용 GitHub의 Private 저장소는 무료로 사용할 수 있어요. GitHub Pages도 무료예요. Google Apps Script는 개인 사용량에선 무료예요. Slack API 사용도 무료예요. 데이터는 개인 Google 계정의 Sheet에 저장되므로 추가 비용 없이 무제한으로 사용할 수 있어요. ## 문제해결 배포나 백엔드 연결에서 문제가 생기면 가이드를 다시 읽어보고, 그래도 해결되지 않으면 Claude에게 문의해 주세요. 에러 메시지나 콘솔 로그 등 문제 상황을 구체적으로 알려주시면 도와드릴게요.
